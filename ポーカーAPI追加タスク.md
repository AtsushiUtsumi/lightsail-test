# ポーカーサーバー
backendでAPIを公開してfrontendからも、curlのようなアクセスできるようにしたいです。

## DDDでいうテーブルクラス設計
- オンラインポーカーのゲームクラス(テーブル単位)を作成する場合、プレイヤーのアクション選択を受け付ける処理を制限時間10秒で受け付けるようにする場合は非同期で受け付けるようにしたい
- テーブルに保持しておく状態変数を列挙してください。誰の手番でポットがいくらかのような情報はテーブルで保持してください。
- テーブル作成時にブラインドやアンティ設定、初期チップ数、最大プレイヤー数、途中入室、途中退出の可否、各プレイヤーの思考時間の制限時間秒数などを設定してインスタンスを作成するようにしてください。
- ポーカーのゲームテーブルをクラスとして定義して、複数のテーブルを同時に起動できるようにそれぞれのテーブルオブジェクトがゲームの状況を進行するようにしてください。
- 各プレイヤーに状況を取得されるときは、他のプレイヤーのハンドの情報は渡さないでください。
- プレイヤーは参加時に自身の情報(ユーザー名、初期チップ数など)
- 各テーブルは参加時に各プレイヤーにトークンを発行してください。
- アクションのログをテーブル単位で保持するようにしてください。get_action_log関数を用意してそのテーブルでプレイされた全てのハンドのログを取得できるようにしてください。
